# Support

## Questions and usage help

Before opening an issue:

1. Check the [README](README.md) and [documentation](docs/).
2. Search existing [issues](https://github.com/xodnr927-byte/repro-evidence-kit/issues).
3. Reduce the problem to a synthetic or clearly redistributable example.

For reproducible bugs, open a
[bug report](https://github.com/xodnr927-byte/repro-evidence-kit/issues/new?template=bug_report.md).
For general-purpose improvements, open a
[feature request](https://github.com/xodnr927-byte/repro-evidence-kit/issues/new?template=feature_request.md).

Support is best effort. Please include the package version, Python version,
operating system, exact command, expected behavior, and observed behavior.

## Sensitive or security-related reports

Do not put credentials, private binaries, proprietary data, sensitive logs, or
private evidence bundles in a public issue.

Report vulnerabilities through
[GitHub private vulnerability reporting](https://github.com/xodnr927-byte/repro-evidence-kit/security/advisories/new).
See [SECURITY.md](SECURITY.md) for the full reporting policy and supported
release line.
